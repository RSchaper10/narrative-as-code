# Service Notes

This starter repo is useful on its own. It is not a gated teaser.

If you want hands-on help adapting the pattern to a real manuscript or studio workflow, the light service layer currently maps to three engagement shapes:

## Narrative System Audit

- fixed-scope review of an existing manuscript workflow
- identifies structure gaps, continuity risk, metadata gaps, and build opportunities
- best when you already have a draft and want a grounded migration plan
- deliverable: written findings plus recommended repo and workflow changes

## Narrative Operating System Setup

- repo setup or migration
- workflow design for prose, canon, metadata, and support files
- build and validation scaffolding
- agent playbook guidance
- best when you want a clean source-of-truth environment quickly
- deliverable: working repo, build path, and operating guidance

## Team Workflow Workshop

- live walkthrough for authors, editors, producers, or narrative teams
- covers repo model, chapter workflow, continuity review, and artifact handoff
- best when a team needs alignment before adopting the system

Use the repo first. Bring in service help only if the migration, cleanup, or team workflow needs extra support.
