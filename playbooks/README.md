# Playbooks

These playbooks turn the starter into an operator-friendly workflow instead of a repo that depends on memory.

Use them when collaborating with Codex, other agents, editors, or teammates who need a crisp, chapter-scoped task.

In a `VS Code + Codex` workflow, these playbooks help keep requests bounded, consistent, and grounded in the right source-of-truth files.

Included workflows:

- `review-chapter.md`
- `refine-chapter.md`
- `check-continuity.md`
- `update-metadata.md`
- `build-editor-packet.md`

For the broader collaboration model, see `docs/codex-vscode-workflow.md`.
