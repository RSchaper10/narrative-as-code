# Revision Plan

## Current Goal

Use the sample to demonstrate how narrative infrastructure supports iteration, not to create a polished novella.

## Next Pass Priorities

1. Keep each chapter clearly tied to a discoverable function in the repository.
2. Preserve a clean distinction between prose, canon, and process notes.
3. Make metadata useful enough that an agent or teammate can orient quickly.
4. Ensure support files feel plausible without bloating the sample.

## Workflow Standard

- Draft in `manuscript/chapters/`
- Lock durable truth in `canon/`
- Add unknowns to `canon/open-questions.md`
- Add supporting context to `research/` or `strategy/`
- Rebuild outputs after meaningful changes
