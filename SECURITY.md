# Security Policy

This repository is a local-first writing workflow starter, not a hosted service, but security and trust still matter.

## Reporting A Vulnerability

If you discover a vulnerability that could affect users of this repository:

- do not open a public issue with exploit details
- contact the repository owner privately through GitHub first
- include the affected file or script, the impact, and the smallest reproduction you can provide

## Scope

Areas most likely to matter here:

- shell scripts
- file-writing utilities
- import and build flows
- GitHub Actions configuration

## What To Expect

Reports will be triaged and validated, then fixed or mitigated as quickly as practical. Public disclosure should wait until the maintainer confirms the issue has been addressed or assessed.
